003

Operating principles

  1. i.

    Write less. The cheapest code to operate is the code that was never shipped. Scope is a design tool.

  2. ii.

    Boring where it counts. Durable primitives — Postgres, message queues, clear interfaces — beat novelty on the critical path.

  3. iii.

    Measure, don’t assume. Traces, benchmarks, and real workloads before opinions.

  4. iv.

    Leave it better. Hand-off matters. Documentation, tests, and the quiet kind of polish other engineers notice six months later.
